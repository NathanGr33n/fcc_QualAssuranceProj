Issue Tracker

User Stories

  1. You can POST /api/issues/{projectname} with form data containing required fields (issue_title, issue_text, created_by) and optional fields (assigned_to, status_text).
  2. The POST request will return the created object with all fields including _id, created_on, updated_on, and open (default true).
  3. If required fields are missing, the response will be { error: 'required field(s) missing' }.
  4. You can GET /api/issues/{projectname} for an array of all issues for that project.
  5. You can filter requests by passing query parameters (e.g., /api/issues/{project}?open=false).
  6. You can PUT /api/issues/{projectname} with an _id and fields to update.
  7. You can DELETE /api/issues/{projectname} with an _id to delete an issue.

Example Usage

GET Examples

/api/issues/{project}
/api/issues/{project}?open=true&assigned_to=Joe
